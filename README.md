# UI Component Lab

> 一个给刚入行PM用的「组件认知」小工具：分不清"这个UI叫什么"的时候，切到查询模式，鼠标停哪儿就告诉你答案。

**在线体验：** https://gongruoyuan.github.io/ui-component-lab/

---

## 为什么做这个

刚入职场做PM，跟设计、研发对接的时候总会卡在一个很小但很尴尬的问题上——

> "就是……那个，页面上那个可以点的、圆圆的、带图标的那个东西"

对方秒懂你说的是"胶囊按钮"还是"图标按钮"还是别的什么，但你说不出这个词，沟通效率就卡在命名这一层。这个工具就是为了解决这一件事：**把"看得懂但叫不出名字"的组件，变成可以直接查的东西**。

## 怎么用

页面顶部有两个模式切换：

- **🖱 操作模式** —— 就是正常页面，能点、能跳转、能感知交互逻辑，跟平时用产品没区别
- **🔍 查询模式** —— 鼠标停在任意组件上，会弹出这个组件的名称和说明（💡 点击查看组件详情）

先用操作模式把页面当真实产品逛一遍，遇到叫不出名字的东西，切到查询模式悬停一下就知道答案了。

## 目前的载体

用一个仿真的示例页面（「好物商城」）做演示场景，尽量贴近日常对接里会遇到的真实页面结构，而不是一个孤立的组件展示墙——组件在具体页面上下文里长什么样、叫什么名字，比脱离场景背单词更容易记住。

## 谁适合用

- 刚接触产品/UI设计的新人PM，需要快速补齐组件命名词汇量
- 想在评审/对接前预先核对术语，减少来回确认成本的人

## 反馈

这是一个自己动手解决真实困扰的小工具，还在持续调整，如果你也遇到过同样的命名困扰，欢迎试用后告诉我哪些地方还没覆盖到。
